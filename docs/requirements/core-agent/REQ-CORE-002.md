## 1. 背景与目标
r-man 需要在跨会话的任务中保持连续性。长期记忆系统通过向量化摘要提供参考。为了节省 Token 并提高交互的确定性，系统采用“纯工具驱动”模式。

---

## 2. 功能需求

### FR-001: 环境初始化与检测 (Bootstrap)
- **依赖检测**: 系统启动时检测 `sqlite-vec` 扩展。
- **异常引导**: 若不可用，输出安装指引并禁用记忆功能。

### FR-002: 手动存入 (Manual Memory Dump)
- **工具调用**: 提供 `memory_dump` 工具。
- **触发逻辑**: 当 Agent 识别到重要知识、任务结论或用户明确要求的偏好时，主动调用。
- **强制脱敏**: 存入前必须替换所有 Key、密码、Token 为 `[REDACTED]`。

### FR-003: 主动检索 (Active Search)
- **工具调用**: 提供 `memory_search` 工具。
- **触发逻辑**: 当用户提问涉及历史模糊背景或 Agent 需要参考之前约定的风格/结论时，主动调用。
- **匹配算法**: 使用 Cosine Similarity (余弦相似度) 计算。

### FR-004: 过期管理 (TTL)
- **字段支持**: 每条记忆必须包含 `expires_at` 字段。
- **默认策略**: 默认有效期为 30 天（自存入起算）。
- **清理逻辑**: 
    - 系统启动时自动扫描并物理删除所有 `expires_at` 小于当前时间的记录。
    - 删除元数据的同时，必须同步清理向量虚拟表中的对应数据。
- **隐私保护**: 严禁存储原始对话序列。
- **性能**: 单次检索响应 < 500ms。

---

## 4. 验收标准
- **AC-001**: 启动自检逻辑正常工作。
- **AC-002**: Agent 能通过 `memory_dump` 存入事实。
- **AC-003**: Agent 能通过 `memory_search` 找回并回答关于历史的问题。
