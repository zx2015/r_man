# REQ-AUDIT-001: 审计日志系统需求

| :--- | :--- | :--- | :--- |

## 1. 业务目标
为 R-MAN 的所有敏感操作建立不可篡改的本地审计链，确保在发生系统变更时可溯源。

## 2. 功能需求

### FR-001: 审计触发范围
所有具有“副作用”的工具调用必须被审计，包括：
- `write_file`: 记录路径、内容摘要、意图。
- `replace`: 记录路径、替换内容、`instruction`。
- `run_shell_command`: 记录完整命令、`description`、执行结果码。
- `process(action=kill)`: 记录被杀 PID。

### FR-002: 存储与轮转策略
- **物理隔离**: 审计日志必须存储在独立于业务日志的文件中。
- **大小控制**: 单个审计文件的大小限制必须可配置（默认 10MB）。
- **历史保留**: 必须实现自动轮转，最多保留最近的 **3 份** 历史审计日志。

### FR-003: 审计数据契约
每条记录必须包含：
- `timestamp`: 精确到毫秒。
- `tool`: 被调用的工具名。
- `intent`: 来自 LLM 产出的执行目的。
- `params`: 实际传入的参数 JSON。
- `status`: 执行结果（Success/Fail）。

## 3. 非功能需求
- **性能影响**: 审计记录过程不得显著阻塞工具执行。
- **持久性**: 审计记录必须在工具执行返回 Observation 之前同步写入磁盘。
