# REQ-CORE-003: 动态 System Prompt 组装系统需求

| 版本号 | 日期 | 变更说明 | 作者 |
| :--- | :--- | :--- | :--- |
| v1.0.0 | 2026-04-16 | 初始版本，定义分段式组装架构与动态注入逻辑 | Gemini CLI |

## 1. 背景与目标
r-man 的行为高度依赖于 System Prompt。为了实现“人设可定制”与“环境感知”，需要一套能够将固定约束、环境状态、工具说明以及用户配置（RMAN.md/TOOLS.md）有机结合的组装系统。

## 2. 功能需求

### FR-001: 分段式组装框架
System Prompt 必须按照以下预定义章节顺序进行组装：

1.  **# 角色和身份**: 包含 R-MAN 的核心愿景与操作准则。
2.  **# 交互格式规范 (Strict Tag Constraint)**: 
    - 强制要求所有回复必须封装在 `<think>` (思考过程) 和 `<final>` (用户回复) 标签中。
    - 禁止在标签外输出任何文本。
3.  **# 工具使用约束**: 
    - **行动优先**: 任务可操作时立即调用工具，不接受纯计划性回复。
    - **简洁风格**: 直接发起工具调用，禁止在回复中描述“我现在要执行...”等冗余信息。
4.  **# 安全 (目录隔离与隐私保护)**: 定义红线，包括工作目录锁定和隐私数据保护。
5.  **# 工具调用风格 (Tool Call Style)**: 
    - 优先采用模型原生的 `tool_calls`。
    - 若不支持，回退至文本模式：工具名必须与列表精确一致，支持单次回复包含完整工具链。
6.  **# Tools**: 动态注入当前已注册的工具列表。
7.  **# 工作目录**: 动态注入当前工作区的绝对路径。
8.  **# 当前日期时间**: 动态注入服务器当前时间与时区。
9.  **# RMAN.md / TOOLS.md**: 动态加载用户自定义内容。

### FR-002: 动态环境注入
- **工作目录**: 必须在运行时获取 `workspace_dir` 的绝对路径并注入。
- **时间信息**: 必须包含日期、具体时间（HH:mm:ss）以及当前时区。

### FR-003: 文件热加载机制
- 每次会话（Session）启动时，必须重新扫描并读取工作区中的 `RMAN.md` 和 `TOOLS.md`。
- 如果工作区文件缺失，系统必须执行“从模板拷贝”的初始化动作。

### FR-004: 工具目录隔离与隐私保护约束
- **目录隔离**: Prompt 中必须明确申明 `read`, `write`, `replace` 工具仅限于操作“工作目录”下的文件。
- **隐私保护**: 强制要求 Agent 严禁泄露任何形式的隐私信息（如 Key、密码、敏感数据）。

### FR-005: 响应美化指令集 (Feishu UI Adaptation)
- **视觉引导**: 必须在 Prompt 中注入专门的章节，指导 LLM 生成适配飞书 `lark_md` 的内容。
- **核心禁令**: 禁止生成 Markdown 表格，禁止使用三级及以下标题（`###`）。
- **色彩注入**: 允许并引导 LLM 使用飞书色彩标签（如 `text_color:green`）来标注关键状态。

## 3. 验收标准
- **AC-001**: 最终生成的 Prompt 包含所有 9 个章节。
- **AC-002**: Prompt 中的“工作目录”段落显示正确的服务器绝对路径。
- **AC-003**: 修改工作区中的 RMAN.md 后，下一轮对话产生的 Prompt 立即包含修改后的内容。
